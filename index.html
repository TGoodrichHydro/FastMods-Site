Why use FastMods?

Update without breaking packs

FastMods checks versions against Modrinth and stages new JARs into a separate /downloads folder before touching your live mods.

Automatic backups

Before replacing anything, FastMods creates a timestamped backup of your existing mods so you can roll back if something feels off.

Zero telemetry

No tracking, no analytics. Only a custom User-Agent for Modrinth API compliance and polite throttling. Your usage stays local.

Fabric loader helper

Quickly install Fabric loader and keep required dependencies like Fabric API in sync with your chosen Minecraft version.

Manual mods supported

For mods not on Modrinth, FastMods can stage existing JARs into the downloads set when they match your target version.

Designed for modded players

Built for people running heavier modpacks who want control, safety, and clear feedback instead of launch-and-pray updates.

How it works

  1. Point FastMods at your mods folder.

    Choose your .minecraft/mods directory and the Minecraft version you’re targeting.

  2. Scan and stage updates.

    FastMods checks each mod against Modrinth and stages compatible updates into a separate downloads folder.

  3. Review & replace.

    Once you’re happy, use Replace Mods to back up your current mods and swap in the staged set.

  4. Launch and play.

    If something fails, restore from the backup folder created automatically before replacement.

FAQ

Does FastMods work with CurseForge/other launchers?

FastMods works at the file level: if it can see a /mods folder, it can scan and stage updates. It doesn’t depend on a specific launcher.

What platforms are supported?

The current build targets Windows with a bundled runtime. Other platforms can run the JAR with a compatible Java + JavaFX setup.

Does it upload my mods anywhere?

No. FastMods only talks to Modrinth’s API to find releases. Your local mods and backups never leave your machine.

Where can I report bugs or request features?

Open an issue on the GitHub repo . Suggestions, bug reports, and weird edge cases are all welcome.
